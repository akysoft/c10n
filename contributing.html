C10N uses gradle build tool for compiling, running tests and building jars, etc. Please get yourself the latest version from the website (tested with 1.3).

For Eclipse users:

Follow these steps to set up c10n in Eclipse:

  1. Checkout the sources. Let's say to /home/rodion/c10n.
  2. Generate Eclipse settings files (.project, .classpath, etc.) by running: $ cd /home/rodion/c10n/master $ gradle eclipse
  3. Import the projects into Eclipse:
    1. Choose File > Import > Existing Projects into Workspace
    2. Set Select root directory: to /home/rodion/c10n
    3. Check all projects (core, master, etc.) and click Finish

You can re-generate Eclipse settings files with:

$ gradle cleanEclipse eclipse

and then Refresh all projects in Eclipse to see the changes.

For IntelliJ IDEA (11.1 or newer) users:

You are highly recommended to use the latest version of IntelliJ IDEA as it contains full round-trip support for Gradle-based projects.

  1. Checkout the sources. Let's say to /home/rodion/c10n.
  2. Choose File > New project ... > Import project from external model
  3. Choose Gradle
  4. Point it to /home/rodion/c10n/master/build.gradle
  5. Rest of settings may stay unchanged

To update changes to the dependencies, pull out the JetGradle from the right and click Refresh Gradle Project.

For IntellijJ IDEA (11.0 and earlier) users:

  1. Checkout the sources. Let's say to /home/rodion/c10n.
  2. Generate IDEA settings files (.ipr, .iml, etc.) by running: $ cd /home/rodion/c10n/master $ gradle idea
  3. Choose File > Open Project ...
  4. Point it to the generated /home/rodion/c10n/master/c10n.ipr file
  5. For some versions of IDEA it will attempt to look for core.iml and tools.iml under master/ instead of looking under the project directory. You will need to explicitly change the path to *.iml for each project before clicking Finish.

You can re-generate IntelliJ IDEA settings files with:

$ gradle cleanIdea idea

This may cause IDEA to prompt you asking to reload the project.

How to build

To build and test the core project:

$ cd core
$ gradle build

To create jars (omit the jar tasks that you don't need):

$ cd core
$ gradle jar sourcesJar guiceJar guiceSourcesJar

To update the maven repo (omit uploadGuiceArchives to skip updating core-ext-guice.jar):

$ cd core
$ gradle uploadArchives uploadGuiceArchives
